# Dive In

A research map for anything you're curious about — install it like an app,
tune its voice to how you learn, and watch your exploration turn into an
actual visual map of what you know.

## What's in this version

- **Neomorphic UI, light + dark.** Toggle in the top bar (sun/moon icon),
  persists across visits.
- **Responsive across phone, tablet, and monitor** — same single codebase.
- **Installable as an app (PWA)** — see "Getting it on your phone" below.
- **AI Summary** — generated by whichever AI connection you've set active;
  falls back to the Wikipedia extract if none is set.
- **Tone & Expertise tuning** (Settings → Tone & Expertise tab): pick a preset
  tone (Direct & Analytical / Friendly Mentor / Academic & Formal) or write
  your own, plus a 1–5 expertise slider that shifts every AI Summary and
  paper insight from "simple language + examples" up to "dense technical
  vocabulary, no hand-holding."
- **Visuals** — a horizontally scrollable strip of images (Wikipedia +
  Openverse, up to ~10), not just three static thumbnails.
- **Primary Sources** — CrossRef + Semantic Scholar, merged, deduped, sorted
  by citation count, scrollable. Click a paper for an AI-generated bottom
  sheet (what it's about, key findings, anything surprising); click again to
  retract.
- **Explore Next, actually populated** — when you have an AI connection set,
  this now asks it directly for 6–8 concrete subtopics/prerequisites to learn
  next to go deeper on the current topic. Falls back to Wikipedia's
  related-pages API when there's no AI connection (which is thinner and is
  why it used to come up empty a lot).
- **Mind Map with learning clusters** — every topic you open becomes a node
  (teal = explored, indigo = suggested). Topics that branched from the same
  exploration thread now cluster into a soft labeled blob — double-click a
  cluster's label to rename it to whatever skill it represents.
- **AI Connections** — add multiple providers (Gemini, Anthropic, or any
  OpenAI-compatible endpoint like Groq/OpenRouter), test each one, see a
  live status dot on the gear icon.

No backend, no build step. Runs as static files.

## Run it

Open `index.html` directly, or serve it statically:

```bash
python3 -m http.server 8000
```

## Deploy it free (GitHub Pages)

1. Push this repo to GitHub.
2. Repo Settings → Pages → Source: `main` branch, root folder.
3. Live at `https://<your-username>.github.io/<repo-name>/`.

**Test on the deployed URL**, not a sandboxed preview — some sandboxed
iframes restrict cross-origin fetches, so panels can look broken there even
when the code is correct.

**About office/corporate networks:** if your workplace blocks AI-model
traffic at the network level, that's a network policy, not a bug in this
tool — no code change can route around a firewall rule. It'll work fine on
any network that isn't specifically blocking those API domains (your phone
on mobile data or home wifi, for instance).

## Getting it on your phone (installable app)

This is a PWA (Progressive Web App) — Android's real, built-in mechanism for
installing a web app so it behaves like a native one: home-screen icon, own
window, works offline for the shell.

1. Deploy to GitHub Pages (above).
2. Open the Pages URL in Chrome on Android.
3. Tap the **⋮** menu → **"Add to Home screen"** / **"Install app"**.
4. It now opens full-screen from your home screen like any other app.

### If you want an actual `.apk` file

You don't need to write native code for this. **[PWABuilder](https://www.pwabuilder.com)**
(free, made by Microsoft) takes any PWA URL and packages it into a real
installable Android package:

1. Deploy to GitHub Pages.
2. Go to pwabuilder.com, paste your Pages URL.
3. It scores your manifest/service worker (already included here) and gives
   you a **"Package for Store"** → **Android** option.
4. Download the generated `.apk`/`.aab`, install directly or side-load it.

This is the honest path — a real compiled APK needs the Android build
toolchain, which isn't something I can run in this chat environment. PWABuilder
does that compilation for you, for free, from the PWA files already in this repo.

## AI Connections

Settings → AI Connections. Add as many as you want, from different
providers, each with its own nickname. **Test** any connection right there —
a status dot turns green (working) or red (failed) after a real test call.
The gear icon carries the same dot for whichever connection is active.

| Provider | Cost | Notes |
|---|---|---|
| Google AI Studio (Gemini) | Free tier | Easiest first connection |
| Anthropic (Claude) | Pay-as-you-go | console.anthropic.com |
| Any OpenAI-compatible endpoint | Varies | Groq, OpenRouter, Together, local Ollama, etc. |

If a connection fails with a browser-level error ("Failed to fetch"), that
usually means the request never reached the provider — a CORS block, an
ad-blocker, or a network policy (see the office-network note above), not a
bad key. Anthropic, Gemini, and most OpenAI-compatible routers (Groq,
OpenRouter) work directly from the browser.

## Tone & Expertise

Settings → Tone & Expertise. Applies to every AI Summary and every paper
insight from that point on:

- **Tone**: Direct & Analytical (default), Friendly Mentor, Academic &
  Formal, or Custom (free-text prompt — describe the voice you want).
- **Expertise (1–5)**: shifts vocabulary and depth. 1 = simple language,
  analogies, worked examples. 5 = dense technical vocabulary, deep insight,
  no basic explanation.

## Mind Map clusters

Every explored topic starts its own cluster. When you follow an "Explore
Next" suggestion or click a related node in the map itself, the new topic
inherits its parent's cluster — so a chain of exploration that started at
"Large Language Models" and wandered through "attention mechanism," "token
embeddings," etc. all group into one blob. Double-click a cluster's label in
the map to rename it — e.g. to the skill you feel you've actually built
("transformer internals").

## How it works (file map)

- `manifest.json`, `service-worker.js`, `icons/` — PWA install support.
- `js/topics.js` — curated shelf of topics for "surprise me" and search
  typeahead.
- `js/connections.js` — multi-provider AI connection manager: storage, the
  provider adapters (Anthropic/Gemini/OpenAI-compatible), test-connection
  flow, gear-icon status dot, the Connections settings pane. Exposes
  `window.NodewayAI`.
- `js/app.js` — dark mode, tone/expertise tuning (`buildStyleInstruction()`
  is prepended to every AI prompt), Wikipedia primer/images, AI-generated
  "Explore Next" with Wikipedia fallback, CrossRef + Semantic Scholar papers
  (sorted, scrollable), the paper insight bottom sheet, search suggestions,
  the surprise-me popover, and the catalogue.
- `js/mindmap.js` — persisted graph (`diveIn.graph` in localStorage) with
  cluster IDs, rendered with D3 (`forceSimulation` + a custom clustering
  force + `d3.polygonHull` for the blob outlines). Exposes `window.NodewayMap`.
